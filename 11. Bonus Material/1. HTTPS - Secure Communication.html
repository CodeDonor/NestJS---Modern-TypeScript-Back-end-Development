Note regarding HTTPS

If you plan to go to production with any NestJS application, or any application at all, make sure to introduce HTTPS.


Why?

When you communicate via HTTP, the data transferred over the network is available for attackers to sniff. If there is an attacker sniffing your network, he/she will be able to, for example, read your plain-text passwords or get the access token from your request.

When communicating over HTTPS, the data transferred is encrypted and only the server is able to decrypt it using a private key.

How?

Setting up HTTPS with a certificate is very easy these days and can especially be reliably and easily done within Amazon Web Services, or any other major trusted authority.
